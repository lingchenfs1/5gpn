# 5gpn · 企业级智能组网网关(自托管 / 开源)

> 一台服务器,把「DNS 解析 + 按业务分流 + 多出口线路 + 运维监控」整合到网关侧;终端零配置,只设一个 DNS 即可纳管。面向企业内网、多站点互联,以及已获授权的跨境专线互通等场景。

**开源仓库(MIT):** https://github.com/lingchenfs1/5gpn

---

## 一、它是什么

5gpn 是一套**自托管的智能组网网关软件**。传统多线路、多站点组网通常要在每台终端装客户端、各自配规则,设备一多就难以维护。5gpn 把这些**策略集中到服务器侧**:

- 终端**只配置一个 DoT(DNS over TLS)**,不安装任何客户端;
- 网关依据企业策略,把**不同业务域名的流量分配到不同的出口线路**(多出口策略路由);
- 转发层基于 **SNI 透明转发**,全程不解密 TLS,性能高、可观测。

一套配置即可纳管 Android / iOS / 电视盒子 / 打印机 / IoT 等所有支持自定义 DNS 的设备:Android 填「私人 DNS」、iOS 扫码装描述文件、其它设备改 DNS 即可。

## 二、架构概览

```
终端设备 (仅配置 DoT)
      │
      ▼
   智能 DNS 网关 ── 按策略解析 ──►  本地直出 / 出口线路 A / 出口线路 B …
   (dnsdist)         │
                     ├─ 国内业务域名  → 国内 DNS 并发竞速,正常直连
                     └─ 指定业务域名  → 引导至网关,按域名分流到选定出口
                                         (SNI / QUIC 透明转发)
```

核心理念:**解析即策略**。终端发出的 DNS 请求到达网关后,网关按规则决定这条流量该走哪条出口线路,终端无感知、零改造。

## 三、核心组件

| 组件 | 职责 |
|------|------|
| dnsdist(智能 DNS + DoT) | 解析入口,按来源与域名策略分流;支持 DoT 加密 |
| 国内 DNS 并发竞速 | 同时查询多个国内公共 DNS,择优返回,兼顾速度与稳定 |
| SNI 透明转发(TCP) | 按 SNI 转发 80/443,不解密 TLS |
| QUIC 透明转发(UDP) | 解析 QUIC/HTTP3 的 SNI,补齐 UDP 443 转发 |
| 出口路由层 | 以策略路由把指定流量导入选定出口线路,可一键切换 |
| 控制层 | 网页控制台 + HTTP API + Telegram 运维 Bot,三端同源同步 |

## 四、主要功能

**智能分流**
- 按域名/规则把不同业务流量分配到不同出口线路或本地直出;
- 规则组(分类)可新建、改名、删除;单条规则可增、删、改、过滤;
- **域名路由测试器**:输入域名,直接看它会经由哪条线路;
- **一键定向**:把某个域名快速定向到指定出口线路,自动处理相关配置。

**多出口管理**
- 多条出口线路统一接入,可一键切换、添加 / 删除;
- **连通性检查 + 实时延迟测试**(优先 ICMP,受限环境自动改用 TCP 探测);
- 每条出口线路独立监测,便于按质量挑选。

**可观测性**
- 概览仪表盘:CPU / 内存 / 硬盘 / 运行时间 + 各服务在线状态;
- **实时上下行速率与连接数**(可开启自动刷新);
- **24 小时流量趋势 / 出口延迟趋势**曲线,鼠标悬浮即显示该时刻各线路数值。

**配置与运维**
- **AI 辅助配置**:绑定你自己的 AI 接口,用自然语言描述意图自动生成分流方案,**人工确认后才生效**;
- **配置备份 / 恢复**:一键导出与还原;
- **Telegram 运维 Bot**:状态查看、切换出口、更新规则、续期证书、重启服务等,仅白名单可操作;
- 网页控制台支持**白天 / 黑夜主题**,并完整适配手机端。

**部署友好**
- 安装流程自动完成域名校验、证书申请、组件编译与系统网络调优;
- 自带低内存模式,**512MB 级别的小机也能稳定运行**。

## 五、网页控制台

可选启用一套带 **HTTPS + 令牌鉴权** 的网页控制台,界面类似企业网关后台:左侧导航(概览 / 出口 / 智能分流 / 控制台),右侧仪表盘与操作区。状态、流量、延迟、出口、分流规则、备份恢复都能在网页上完成,且与 Telegram Bot 实时同步。控制台是纯静态页面,可托管在任意位置,填入网关地址与令牌即可使用。

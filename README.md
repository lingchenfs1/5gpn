# 5gpn 企业级智能组网网关 · 介绍与上手教程

> 一台服务器,把「DNS 解析 + 按业务分流 + 多出口线路 + 运维监控」整合到网关侧;终端零配置,只设一个 DNS 即可纳管。面向企业内网、多站点互联,以及已获授权的跨境专线互通等场景。

**开源仓库(MIT):** https://github.com/lingchenfs1/5gpn

> 适用范围:仅用于**已获合法授权**的企业组网与技术研究。请在符合所在地法律法规及运营商相关规定的前提下部署使用。

---

# 第一部分:产品介绍

## 一、它是什么

5gpn 是一套**自托管的智能组网网关软件**。传统多线路、多站点组网通常要在每台终端装客户端、各自配规则,设备一多就难维护。5gpn 把这些**策略集中到服务器侧**:

- 终端**只配置一个 DoT(DNS over TLS)**,不安装任何客户端;
- 网关依据企业策略,把**不同业务域名的流量分配到不同出口线路**(多出口策略路由);
- 转发层基于 **SNI 透明转发**,全程不解密 TLS,性能高、可观测。

一套配置即可纳管 Android / iOS / 电视盒子 / 打印机 / IoT 等所有支持自定义 DNS 的设备。

## 二、架构概览

```
终端设备 (仅配置 DoT)
      │
      ▼
   智能 DNS 网关 ── 按策略解析 ──►  本地直出 / 出口线路 A / 出口线路 B …
   (dnsdist)         │
                     ├─ 国内业务域名  → 国内 DNS 并发竞速,正常直连
                     └─ 指定业务域名  → 引导至网关,按域名分流到选定出口
                                         (SNI / QUIC 透明转发)
```

核心理念:**解析即策略**。终端的 DNS 请求到达网关后,网关按规则决定这条流量走哪条出口线路,终端无感知、零改造。

## 三、核心组件

| 组件 | 职责 |
|------|------|
| dnsdist(智能 DNS + DoT) | 解析入口,按来源与域名策略分流;支持 DoT 加密 |
| 国内 DNS 并发竞速 | 同时查询多个国内公共 DNS,择优返回,兼顾速度与稳定 |
| SNI 透明转发(TCP) | 按 SNI 转发 80/443,不解密 TLS |
| QUIC 透明转发(UDP) | 解析 QUIC/HTTP3 的 SNI,补齐 UDP 443 转发 |
| 出口路由层 | 以策略路由把指定流量导入选定出口线路,可一键切换 |
| 控制层 | 网页控制台 + HTTP API + Telegram 运维 Bot,三端同源同步 |

## 四、主要功能

- **智能分流**:按域名/规则把不同业务分配到不同出口线路或本地直出;规则组可新建/改名/删除,单条规则可增删改与过滤;内置**域名路由测试器**与**一键定向**。
- **多出口管理**:多条出口线路统一接入、一键切换、增删;连通性检查 + **实时延迟测试**(优先 ICMP,受限环境自动改用 TCP 探测)。
- **可观测性**:CPU/内存/硬盘/运行时间、**实时上下行速率与连接数**、**24 小时流量与出口延迟曲线**(悬浮显示各线路数值)。
- **AI 辅助配置**:绑定你自己的 AI 接口,用自然语言描述意图自动生成分流方案,**人工确认后才生效**。
- **运维便利**:配置一键备份/恢复;Telegram 运维 Bot(白名单);网页控制台支持白天/黑夜主题并完整适配手机端。
- **部署友好**:自动完成域名校验、证书申请与系统调优;自带低内存模式,**512MB 小机也能稳定运行**。

# 第二部分:上手教程

按以下步骤完成后,即可在云服务器上部署 5gpn,并在终端配置使用。

## 准备工作

- 一台有**公网 IPv4** 的云服务器(内存 512MB 起);
- 一个**你能自行管理解析的域名**(已有域名即可;没有可在免费 DNS 服务商处申请,如 ClouDNS https://www.cloudns.net );
- 服务器的 **SSH 登录信息**;
- (可选)Telegram Bot Token,用于手机端运维;
- (可选)你自己的 AI 接口,用于 AI 辅助生成分流规则。

## 第 1 步:准备云服务器

准备一台有公网 IP 的云服务器,记录其**公网 IP** 与 SSH 登录方式。支持 Ubuntu / Debian / CentOS · Rocky · AlmaLinux / RHEL / Fedora,x86_64 或 ARM64。

## 第 2 步:把域名解析到服务器

在 DNS 服务商处给一个域名(裸域或子域,如 `gw.example.com`)添加一条 **A 记录**指向服务器公网 IP。建议把 TTL 调小(60–300 秒)以便快速生效。

> 解析刚创建可能需几分钟生效,属正常现象。

## 第 3 步:SSH 登录并一键安装

SSH 登录服务器后执行:

```bash
curl -fsSL https://raw.githubusercontent.com/lingchenfs1/5gpn/main/quick-install.sh -o /tmp/5gpn.sh && sudo bash /tmp/5gpn.sh
```

脚本会自动检测系统、安装依赖,并**交互提示输入你自己的域名**(填第 2 步的域名)。

## 第 4 步:验证解析并自动申请证书

脚本会显示需解析到的服务器 IP,并**轮询验证 A 记录**(最长约 120 秒);通过后自动申请并配置证书。若尚未生效会自动重试——确认域名与 IP 一致即可。

## 第 5 步:查看部署结果

完成后会显示部署信息:**DoT 地址**、DNS 查询地址、转发服务端口,以及 **iOS 描述文件下载地址**(及二维码)。记下 DoT 地址备用。

## 第 6 步:配置客户端(终端只配一个 DNS)

- **Android**:设置 → 网络和互联网 → 私人 DNS → 选"指定的私人 DNS 服务商主机名" → 填入第 5 步的**域名**;
- **iOS / iPadOS**:用相机扫描脚本输出的二维码安装描述文件,或把描述文件链接复制到 **Safari** 打开安装;
- **电视盒子 / 打印机 / IoT**:在系统网络设置里把 DNS 改为该地址。

配置完成即可使用。

## 第 7 步(可选):启用网页控制台

```bash
cd /opt/5gpn && sudo ./install.sh --setup-api
```

执行后会打印**控制台地址**与**访问令牌**。把仓库里的 `webui/index.html` 托管到任意位置(或本地打开),填入地址和令牌即可使用。安装主流程中也会询问是否一并启用。

## 第 8 步(可选):启用 Telegram 运维 Bot

```bash
cd /opt/5gpn && sudo ./install.sh --setup-tgbot
```

按提示填入 Bot Token 与授权 ID,即可在手机上查看状态、切换出口、更新规则、续期证书等(仅白名单可操作)。

# 注意事项

- 运行脚本前确认服务器能正常访问 GitHub 的 raw 地址;
- 域名 A 记录需提前指向服务器 IP,刚创建可能需几分钟生效;
- 证书申请依赖解析正确,请确保 IP 填写一致;
- 512MB 小内存机会自动开启低内存模式,正常现象。

# 常见问题

**1. 域名解析验证一直不通过?** 检查 A 记录域名与 IP 是否与脚本提示一致;刚创建稍等几分钟再试,或调小 TTL。

**2. 安卓端私人 DNS 填什么?** 填部署完成后显示的**域名**(即第 2 步解析好的那个),不是 IP。

**3. iOS 端怎么配置?** 扫描脚本输出的二维码安装描述文件,或把描述文件链接复制到 Safari 打开安装。

**4. 网页控制台连不上?** 确认已执行 `--setup-api`,地址与令牌填写正确,且服务器对应端口可访问。

**5. 想换访问令牌?** 修改服务器上控制台配置中的令牌后,重启对应服务即可。

# 5gpn 上手教程:一台服务器搭好企业级智能组网网关(终端零配置)

写个详细的上手教程。**5gpn** 是一套自托管、开源的企业级智能组网网关:把「DNS 解析 + 按业务分流 + 多出口线路 + 运维监控」都放到服务器侧,终端只配一个 DNS 就能纳管,适合企业内网、多站点互联,以及已获授权的跨境专线互通等场景。

仓库(MIT 开源):https://github.com/lingchenfs1/5gpn

> 合规声明:仅面向已获合法授权的企业组网与技术研究用途,请在符合所在地法律法规及运营商规定的前提下部署使用。

---

## 一、准备工作

开工前先备齐三样:

1. **一台服务器**:公网 IPv4,内存 512MB 起(自带低内存模式,小机也能跑)。
2. **一个域名**:你能自行管理解析的域名(裸域或子域都行),用来给终端做加密 DNS。
3. **系统**:Ubuntu 20.04+ / Debian 11+ / CentOS·Rocky·Alma / RHEL / Fedora,x86_64 或 ARM64,`root` 权限。

提前把域名的 **A 记录**指向服务器公网 IP(TTL 设小一点,如 60–300,生效快)。

## 二、一键部署

SSH 登录服务器,执行:

```bash
curl -fsSL https://raw.githubusercontent.com/lingchenfs1/5gpn/main/quick-install.sh -o /tmp/5gpn.sh && sudo bash /tmp/5gpn.sh
```

脚本会全自动走完:依赖安装 → 交互输入你的域名 → 校验解析 → 申请证书 → 编译安装各组件 → 系统网络调优 → 启动服务。

过程中会**依次询问两个可选项**(都可留空跳过,以后再开):

- 是否启用 **Telegram 运维 Bot**;
- 是否启用 **HTTP API + 网页控制台**。

装完终端就能用了。常用管理命令:

```bash
sudo ./install.sh --status         # 运行状态 + 当前出口线路
sudo ./install.sh --list-exits     # 列出所有出口线路
sudo ./install.sh --setup-api      # 启用网页控制台(可选)
sudo ./install.sh --setup-tgbot    # 启用 Telegram 运维(可选)
```

## 三、终端接入(零客户端)

终端**只需要配置一个加密 DNS(DoT)**,指向你的域名即可,不用装任何客户端:

- **Android**:设置 → 网络和互联网 → 私人 DNS → 填你的域名;
- **iOS / iPadOS**:安装过程会生成一个 DNS over TLS 描述文件并输出二维码,扫码安装即可(仅在蜂窝网络下启用,连 Wi‑Fi 自动停用,不影响家庭/办公 Wi‑Fi);
- **Windows / macOS / Linux**:在系统 DNS 设置里启用 DoT,或用本地 DoT 转发器指向域名;
- **电视盒子 / 打印机 / IoT**:把 DNS 改成网关地址即可纳管。

一套配置纳管所有支持自定义 DNS 的设备,新设备入网几秒搞定。

## 四、启用网页控制台

执行 `sudo ./install.sh --setup-api` 后,会打印 **网页地址** 与 **访问令牌**(在 Telegram Bot 里发 `/api` 也能随时查看)。

网页控制台是一个**纯静态单页**(仓库里的 `webui/index.html`),怎么放都行:

- 最快:把 `index.html` 下载下来双击打开;
- 推荐:托管到 Cloudflare Pages / 任意静态托管(自动 HTTPS、可自动随仓库更新);
- 或用自己的 Nginx + 证书托管。

> 详细托管步骤见仓库 `webui/README.md`。

打开后填入「网页地址 + 令牌」即可连接,令牌存在浏览器本地,下次自动登录。控制台与 Telegram Bot **实时同步**,改任意一边另一边立刻可见。

## 五、日常使用

控制台是企业网关后台风格:左侧导航(概览 / 出口 / 智能分流 / 控制台),右侧仪表盘与操作区。

### 1. 概览
- 资源横条:CPU / 内存 / 硬盘 / 运行时间 + 各服务在线状态;
- **实时上下行速率与连接数**(可开「自动刷新」);
- **24 小时流量趋势 / 出口延迟趋势** 曲线,鼠标悬浮即显示该时刻每条线路的数值。

### 2. 出口线路管理
- 多条出口线路统一接入,**一键切换**、添加 / 删除;
- 进「出口」→ 添加 → 填一个名称(如 `lineA`)→ 按提示粘贴这条线路的接入配置即可;
- **连通性检查 + 实时延迟测试**(优先 ICMP,受限时自动改用 TCP 探测),按质量挑线路。

### 3. 智能分流
- 按域名/规则把不同业务分配到不同出口线路或本地直出;
- **规则组**可新建 / 改名 / 删除;**单条规则**可增 / 删 / 改 / 过滤;
- **域名路由测试器**:输入一个域名,直接看它会经由哪条线路;
- **一键定向**:把某个域名快速定向到指定出口线路,相关配置自动处理。

### 4. AI 辅助配置(可选)
- 在「智能分流」里绑定**你自己的 AI 接口**(填 Base URL / Key / 模型);
- 用自然语言描述意图,自动生成一份分流方案;
- **方案必须人工确认后才会生效**,不会自动改动。

### 5. 备份与运维
- **配置备份 / 恢复**:一键导出全部配置为文件,重装后一键还原;
- **Telegram Bot**:状态、切换出口、更新规则、续期证书、重启服务、出 iOS 二维码等,仅白名单 ID 可操作;
- 网页支持**白天 / 黑夜主题**,并完整适配手机端。

## 六、常见问题

**Q:终端要装客户端吗?**
不用。只配一个 DoT DNS 即可,所有支持自定义 DNS 的设备都能纳管。

**Q:证书会自动续期吗?**
会。安装时已配置自动续期与定时任务,无需手动维护。

**Q:某个域名没有按预期走对线路?**
用控制台的「域名路由测试器」查一下它实际会经由哪条线路;需要的话用「一键定向」把它指到目标线路。

**Q:网页面板放在外网安全吗?**
页面本身不含任何密钥,真正的钥匙是访问令牌。请只走 HTTPS、妥善保管令牌;想换令牌改服务器上 `api.env` 后重启服务即可。

**Q:小内存机够用吗?**
512MB 即可,安装时会自动开启低内存模式(缩缓存、限内存、建 swap 等)。

## 七、结语

整套体验下来,最舒服的是**终端零配置 + 策略集中在网关**:一套 DNS 纳管所有设备,多线路按业务分流、网页上点一下就改,还有实时速率、连接数、24h 流量/延迟曲线可观测,网页 + Telegram 双端同步运维。
